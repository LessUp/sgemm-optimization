# Research Desk

The research desk is where the project stops speaking only in its own voice. It points outward, to the papers, repositories, and historical decisions that make the whitepaper stronger.

## What belongs here

| Surface | Why it matters |
|---|---|
| [Curated References](./references) | Canonical documentation, papers, tools, and codebases |
| [Related Projects](./related-projects) | Comparative context, what this repository is and is not |
| [Evolution Notes](./evolution) | Why the public narrative and route map changed |

| [Further Reading](./further-reading) | Recommended next routes after each section |

## Research questions this section answers

1. Which official sources justify the architectural claims?
2. Which public SGEMM or GEMM projects are worth comparing against?
3. Which papers improve the mental model behind the ladder?
4. How did this project evolve from a good repo into a stronger public artifact?

## Suggested research path

1. Start with [Curated References](./references)
2. Compare with [Related Projects](./related-projects)
3. Read [Evolution Notes](./evolution)
4. Continue into [Papers](./papers) or [Further Reading](./further-reading)
