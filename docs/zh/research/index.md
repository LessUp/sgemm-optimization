# 研究总览

研究资料台是这个项目停止只用自己声音说话的地方。它向外指向——指向那些让白皮书更有力、更诚实的论文、仓库和历史决策。

## 为什么需要研究资料台

一个没有外部参考的优化项目，不过是一个孤立的论点。研究资料台回答了四个任何内部页面单独都无法回答的问题：

1. 哪些官方来源为架构章节中的论断提供支撑？
2. 哪些公开的 SGEMM 或 GEMM 项目展示了更强或不同的方法？
3. 哪些论文改善了阶梯背后的思维模型，尤其是关于 Tensor Core 和内存层次的推理？
4. 这个项目当前的叙事形态是如何从早期更弱的形式演变而来的？

## 推荐研究路径

第一次浏览研究资料台：

1. 从[参考文献地图](./reference-map)开始——了解哪些论断有外部支撑
2. 继续阅读[相关项目](./related-projects)——校准这个项目在领域中的位置
3. 阅读[演进思考](./evolution)——了解为什么选择了当前的呈现形态
4. 使用[论文索引](./papers)或[延伸阅读路线](./further-reading)深入特定主题

## 作为评审者使用研究资料台

如果你在审计白皮书的论断，研究资料台为你提供了检验工具：

- 在架构或验证章节找到论断。
- 在[参考文献地图](./reference-map)中找到其支撑来源。
- 检查该来源是否在所声明的范围内支持该论断。

无法追溯到外部来源、内部实现或明确标注的待决问题的论断，就是应该被质疑的论断。

## 研究资料台不是什么

研究资料台不是一个预先为项目提供正当性依据的文献综述。它是一个活跃的索引，通过使项目的外部依赖可见、可检验，来保持白皮书的诚实。

它也不是竞争分析。[相关项目](./related-projects)页面明确承认了本仓库不做什么——因为了解一个项目的局限性，是理解它的一部分。
