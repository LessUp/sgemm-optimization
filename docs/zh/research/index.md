# 研究总览

研究资料台是项目不再只用自己声音说话的地方。它负责把视线引向论文、仓库和历史决策，让白皮书拥有更强的外部支撑。

## 这里应该包含什么

| 表面 | 为什么重要 |
|---|---|
| [参考资料清单](./references) | 官方文档、论文、工具和代码库的规范入口 |
| [相关开源项目](./related-projects) | 对照语境，说明本仓库是什么，也说明它不是什么 |
| [演进思考](./evolution) | 解释公共叙事和路由结构为什么会演进成现在这样 |
| [论文索引](./papers) | 理论和优化历史的聚焦阅读表 |
| [延伸阅读路线](./further-reading) | 每个章节读完后应该继续去哪里 |
| [性能案例库](./performance-casebook) | 如何理解代表性结果而不过度声称 |

## 这个章节回答的问题

1. 哪些官方资料支撑了架构主张？
2. 哪些公开 SGEMM 或 GEMM 项目值得拿来对照？
3. 哪些论文能补强这条阶梯背后的心智模型？
4. 这个项目是怎样从一个好仓库，继续变成更强的公共作品？

## 建议研究路径

1. 先看 [参考资料清单](./references)
2. 再看 [相关开源项目](./related-projects)
3. 然后阅读 [演进思考](./evolution)
4. 最后进入 [论文索引](./papers) 或 [延伸阅读路线](./further-reading)
