# 演进思考

这个项目同时沿着两条线演进：一条是 kernel 变得更有结构，另一条是公共叙事变得更明确。

## 代码层的演进

kernel 阶梯从可读的 FP32 基线逐步走向更高吞吐策略，同时始终保留可比较的 launcher 契约。这个细节很关键，因为“可比较”本身就是教学价值的一部分。

## 文档层的演进

公共站点从“好的文档”继续推进成更有立场的白皮书：

1. 先把架构和方法论分清楚
2. 再把验证边界和参考资料提升为一级主题
3. 现在进一步重组为导读、架构、学院、验证、研究五个主轴

## 为什么这种形态更好

- 评审者能更快找到方向。
- 候选人得到一条更好的讲述路径。
- 高级读者能直接看到技术谱系和对照材料。
- 站点可以继续变深，而不会继续变乱。
